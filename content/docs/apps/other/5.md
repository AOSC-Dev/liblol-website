### 安装libLoL及该应用

参考[使用方法](/docs/usage)

{{< issue 无法在非原厂支持系统中安装 >}}
官网下载钉钉只适配了特定发行版，在 AOSC OS 上会闪退，需要进行额外配置才能使用。

#### AOSC OS

首先，进入钉钉安装目录。以本页面更新时最新版本`8.1.0.6011301`为例，执行以下操作：
```
cd /opt/apps/com.alibabainc.dingtalk/files/8.1.0-Release.6011301/
```
随后，使用以下命令启动钉钉：

```
./com.alibabainc.dingtalk
```
会提示缺少库文件，产生类似下面的输出：
```
Run Main is_gpu=0is_zygote=0 is_render=0 is_crashpad_handler=0 cmd:./com.alibabainc.dingtalk
Load/opt/apps/com.alibabainc.dingtalk/files/8.1.0-Release.6011301//dingtalk_dll.so failed! Err=liblber-2.4.so.2:cannot open shared object file: No such file or directory
```
发现提示缺少`liblber-2.4.so.2`库文件。

从以下链接下载所需的包：
```
https://pkg.loongnix.cn/loongnix/20/pool/main/o/openldap/libldap-2.4-2_2.4.47%2Bdfsg-3%2Bdeb10u2.lnd.2_loongarch64.deb
```

下载完成后，使用`dpkg -x`命令解压下载的包，从中找到缺失的库文件`liblber-2.4.so.2 `。

将该文件拷贝到以下目录：
```
 /opt/lol/lib/loongarch64-aosc-linux-gnuow/ 
```

完成拷贝后，使用以下命令修改启动脚本：
```
vim  /opt/apps/com.alibabainc.dingtalk/files/Elevator.sh
```
将其中的内容替换为：
```
#!/bin/sh
cd /opt/apps/com.alibabainc.dingtalk/files/8.1.0-Release.6011301/
./com.alibabainc.dingtalk
```
随后，正常打开钉钉即可运行。
